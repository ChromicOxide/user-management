## 项目简介

本项目是一个面向 **Web 安全入门** 的实操教学案例，分七个阶段递进式完成：

| 阶段 | 主题 | 核心内容 |
|:----:|------|---------|
| 阶段一 | 安全审计 | 原始代码 19 漏洞 → 三轮修复 → OWASP Top 10 + WCAG 合规 |
| 阶段二 | SQL 注入攻防 | 新增注册/搜索（故意留洞）→ POC 验证 → 参数化查询修复 |
| 阶段三 | 文件上传治理 | 新增头像上传（无校验）→ 6 个上传漏洞 → 全链路封堵 |
| 阶段四 | 权限提升修复 | 新增个人中心/充值（无校验）→ 7 个越权漏洞 → 权限闭环 |
| 阶段五 | 文件包含治理 | 新增动态页面加载（无校验）→ 5 个文件包含漏洞 → 路径穿越封堵 |
| 阶段六 | 密码修改治理 | 新增密码修改（无校验）→ 4 个CSRF/越权漏洞 → 权限闭环修复 |
| 阶段七 | SSRF 漏洞治理 | 新增 URL 抓取（无校验）→ 7 个 SSRF 漏洞 → 内网协议全链路封堵 |

## 阶段一：原始代码安全审计

原始代码包含 **19 个安全漏洞**（高危 5 / 中危 8 / 低危 6），经三轮修复清零。

**核心修复：**
bcrypt 哈希存储 · CSRF 令牌 · 30 分钟会话过期 · 5 次登录锁定 · 版本指纹隐藏 · 计时攻击防御 · 会话固定修复 · WCAG 2.1 AA 无障碍 · 6 项安全响应头 · 审计日志

## 阶段二：SQL 注入攻防

新增 SQLite 数据库 + 用户注册 + 搜索，使用 f-string 拼接 SQL。

**POC 验证：**
- UNION 注入：`' UNION SELECT 1,'inj','inj@x.com','138'--`
- OR 万能条件：`' OR '1'='1`
- 注册注入：`hacker', 'pass')--`

**修复：** 全量参数化查询 + CSRF 防护 + bcrypt 哈希 + LIKE 通配符转义

## 阶段三：文件上传漏洞治理

新增头像上传功能，完全不校验文件类型，直接保存原始文件名。

**发现的 6 个上传漏洞：** 无限制任意文件上传 · 路径穿越覆盖 · 无 CSRF · 同名覆盖 · SVG XSS · 特殊字符异常

**修复：** ALLOWED_EXTENSIONS 白名单 · safe_filename() 路径过滤 · CSRF 校验 · 唯一文件名 · MAX_CONTENT_LENGTH

## 阶段四：权限提升与业务逻辑修复

新增个人中心（`/profile`）和充值（`/recharge`），不做归属校验。

**发现的 7 个漏洞：** IDOR 越权 · 无需登录 · 负数充值 · 越权充值 · 溢出 · 枚举 · 权限分级缺失

**修复：** @login_required · session 用户归属校验 · amount 正数校验 · MAX_BALANCE 上限

## 阶段五：文件包含漏洞治理

新增动态页面加载功能（`/page?name=help`），直接拼接用户输入到文件路径。

**发现的 5 个漏洞：**

| 编号 | 漏洞名称 | 等级 | 攻击方式 |
|------|---------|:----:|---------|
| FI-01 | 路径穿越读取任意文件 | 🔴 高危 | `/page?name=../app.py` 读取源码 |
| FI-02 | 读取系统敏感文件 | 🔴 高危 | `/page?name=../../../etc/passwd` |
| FI-03 | 未授权访问动态页面 | 🟠 中危 | 清除 Cookie 直接访问任意 page |
| FI-04 | 读取源码泄露凭据 | 🟠 中危 | 提取 secret_key 伪造 session |
| FI-05 | HTML 嵌套渲染异常 | 🟢 低危 | 双层 html 标签导致页面结构异常 |

**修复方案：**
```python
safe_name = name.replace("\\", "/").split("/")[-1]   # 防路径穿越
ALLOWED_PAGE_EXT = {".html"}                           # 扩展名白名单
os.path.realpath() 确保在 pages/ 目录内                # 真实路径校验
@login_required                                        # 登录态控制
```

**修复验证：** 路径穿越、系统文件读取、PHP 文件、未登录访问全部拦截 ✅

## 阶段六：密码修改漏洞治理

新增密码修改功能（`/change-password`），不做任何安全校验。

**发现的 4 个漏洞：**

| 编号 | 漏洞名称 | 等级 | 攻击方式 |
|------|---------|:----:|---------|
| PWD-01 | CSRF 跨站请求伪造 | 🔴 高危 | 恶意页面自动提交表单修改密码 |
| PWD-02 | 越权修改任意用户密码 | 🔴 高危 | alice 可修改 admin 密码 |
| PWD-03 | 无原密码验证 | 🔴 高危 | 直接传入新密码即可修改 |
| PWD-04 | 无确认密码校验 | 🟢 低危 | 误输密码导致无法登录 |

**修复方案：**
```python
# CSRF 防护
<input type="hidden" name="csrf_token" value="{{ csrf_token() }}">
if not token or token != session.get("_csrf_token"): return 400

# 只能修改自己的密码
if form_username != current_user: return 403

# 原密码验证
if not check_password_hash(user_record["password"], old_password): return "原密码错误"

# 确认密码校验
if new_password != confirm_password: return "两次输入的密码不一致"
```

**修复验证：** 无 CSRF 拦截、越权拦截、原密码校验、确认密码校验全部通过 ✅

## 阶段七：SSRF 漏洞治理

新增 URL 抓取功能（`/fetch-url`），直接使用 urllib 访问用户提交的 URL，不做任何检查。

**发现的 7 个漏洞：**

| 编号 | 漏洞名称 | 等级 | 攻击方式 |
|------|---------|:----:|---------|
| SSRF-01 | 无协议白名单 — file:// 读取任意文件 | 🔴 高危 | `file:///etc/passwd` 读取系统文件 |
| SSRF-02 | 无内网 IP 限制 — 内网服务探测 | 🔴 高危 | `http://127.0.0.1:5000/` 访问内网 |
| SSRF-03 | 云元数据攻击 | 🔴 高危 | `http://169.254.169.254/` 获取云凭证 |
| SSRF-04 | 无端口限制 — 内网端口扫描 | 🟠 中危 | 遍历内网 IP 端口探测服务 |
| SSRF-07 | 无重定向限制 — 302 跳转绕过 | 🟠 中危 | 外部 302 跳转到 file:// |
| SSRF-05 | 无返回值限制 | 🟢 低危 | 内网响应内容泄露 |
| SSRF-06 | 无 URL 长度限制 | 🟢 低危 | 超长 URL 日志注入 |

**修复方案：**
```python
ALLOWED_PROTOCOLS = {"http", "https"}                       # 协议白名单
_is_internal_ip(host)                                        # 内网 IP 检测
NoRedirectHandler()                                          # 禁止 302 跟随
len(target_url) > 2048                                       # 长度限制
```

**修复验证：** file 协议拦截、内网 IP 拦截、云元数据拦截、302跳转拦截全部通过 ✅
